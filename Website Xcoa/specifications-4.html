The IoT City Data Model.

 The IoT City Data Model contains information about the sensors being used by the platform, as well as its location, status (active/inactive), the sensor type (temperature/air conditions/radiation/people counter/noise levels/lighting/waste management) and other information.


 Each sensor can have subscriptions associated. The subscriptions are the streams that will send data from the sensor to the platform. Each subscription can have a subtype (e.g. “Co2 level” or “Pressure level”) and a flag that indicates if it is working (if the server could not fetch any data from the broker, it will set working to “False”). The subscriptions can have values that will be gathered by the server when in contact with the broker.


 All users will be registered on the platform with personal information (username, email, password). Each user can have notes, defined by a name, start and end date. It is also possible to define alarms or rules for each user, the subscription associated, the threshold value and the dates that the alarm is active. An actuator can be triggered by an alarm (e.g. “Every day, since 7 p.m. until 7 a.m., turn on all public illumination”).


 The user does not have access to any subscription by default. Instead, the administrator has the possibility to create groups to aggregate subscriptions and each user is only authorized to access the subscriptions that are associated to their groups. Finally, it is possible to the mobile user to report occurrences in the city for the city manager to resolve, indicating the title of the report, the user name, the user e-mail and the attachments to send to the server.
