IoT City

The city in your hands

IoT City is a platform designed for city managers and general users who live in a smart environment.
Its' main focus is to gather, analyze and show various types of data from a specific city and give the managers
the ability to act upon it through a web portal, besides general information for everyone through a mobile app
and another mobile app exclusively for the city manager.


What is IoTCity?


 IoT City project main focus is to allow city managers to monitor and analyse data from a city and act upon it through a web portal.​ The data is gathered from sensors of several types (temperature, radiation, noise, lighting, people counter, air quality and waste containers) distributed by the city, being stored, analysed and displayed to the city manager, who can monitor, set alarms, rules or act on the sensors if possible.​
 The management of all users, sensors and data associated to them is done on a admin portal.​
 The citizen mobile application for the regular citizen shows relevant city sensors’ data and allows to report occurrences on the city directly to the city manager along with photos or videos (e.g. waste container that could be damaged).
 The city manager mobile application is very similar to the citizen mobile app. The difference is that it does not allow to do reports and notifies the city manager when there are triggered alerts such as on the web portal.  This project works in collaboration with Altice Labs, our industrial collaborator, and IT (Instituto de Telecomunicações), providing us a wide range of sensors to work with.

Platform views

Mobile applications


Main dashboard view
On the main dashboard view, we can see the values for each sensor type. Each card will contain values of the nearest sensor of that type.

Menu view
The menu allows the user to access the dashboard, map or different metrics of sensors in just one click!

Map view
The user can easily see the location of the sensors in his city, and tap on one to see its metrics.

Temperature sensors view
When a metric is choosed, it is displayed a chart with the values from the sonsors of that metric.

Exclusive to the city manager mobile application


Alerts view
In the city manager app, the user can see all the triggered alerts in the above screen. He can also dismiss them.

Web portal


Main dashboard page
On the main dashboard page, the user has access to the latest values received from the sensors of each type. He also has information about the triggered alarms, the user reports not seen and the created notes.

Map page
Map with markers that shows the location of the sensors.


Temperature page
Each metric page has information about the sensors of that type and a chart with the received values.

Admin portal


Main admin view
On this view the user has information about all the information gathered by the platform, and can choose which one he wants to change (users, sensors, subscriptions, ...).

Add sensors' page
If the user decides to add a sensor, he can fill in this form and click on "Add".

Sensors list page
On this view the user can list all sensors available in the platform.

Citizen mobile application

Download it now!


Installing our mobile app on your smartphone is a simple and brief process.



First thing you have to do is, in your smartphone, go to Settings, then Security, and then enable installation of apps of unknown sources.

After this, download the apk file through your smartphone. The file will be stored in your "Downloads" folder. Go there and click on the file. You will be prompted, asking if you want to install this app. Accept the instalation and wait until the mobile app is fully installed.

Timeline

  • Milestone 1

    01/03/2017

    Definition of the scope and requirements.

  • Milestone 2

    22/03/2017

    Web portal showing the metrics of one type of sensor at least. The sensor used should send his data to SmartIoT, being retrieved by the server. All data should be saved in data persistence. A local broker should be ready for usage.

  • Milestone 3

    19/04/2017

    Web portal with map, showing the locations of the sensors. At least three sensors should be connected to the platform. Mobile application should have all the views prepared but still not connected with the server.

  • Milestone 4

    03/05/2017

    All sensors should be connected to the platform. Web portal should be functional with all these sensors. The actuators should also be functional. Mobile application should connect to the server and display the information. The monitor and alarms module should be ready, as well as the data persistence. AAA done and working with the platform.

  • Milestone 5

    31/05/2017

    User reports should be working. Final work on mobile and web portal interfaces. API to register sensors and streams directly to the platform. Dynamic localization of sensors. Deployed to a production environment.

  • Milestone 6

    14/06/2017

    Final submission.
